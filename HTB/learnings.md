nmap for portscanning
telnet Unverschlüsselte Remote-Kommandozeile zur Verwaltung, prüft zB ob TCP verbindung steht
FTP file transfer protocol verwendet standardmäßig Port 21 für den Steuerkanal (Befehle) und Port 20 für den Datenkanal (Dateiübertragungen).


Essenzielle Ports sind
80/443 (HTTP/HTTPS), 22 (SSH), 21 (FTP), 53 (DNS) und 3389 (RDP)
    20/21 - FTP (File Transfer Protocol): Dateiübertragungen.
    22 - SSH (Secure Shell): Verschlüsselte Remote-Verbindungen.
    25 - SMTP (Simple Mail Transfer Protocol): E-Mail-Versand.
    53 - DNS (Domain Name System): Namensauflösung.
    80 - HTTP (Hypertext Transfer Protocol): Unverschlüsselter Webverkehr.
    110 - POP3 (Post Office Protocol): E-Mail-Abruf.
    123 - NTP (Network Time Protocol): Zeitsynchronisation.
    143 - IMAP (Internet Message Access Protocol): E-Mail-Verwaltung.
    161/162 - SNMP (Simple Network Management Protocol): Netzwerküberwachung.
    443 - HTTPS (HTTP Secure): Verschlüsselter Webverkehr (SSL/TLS).
    445 - SMB (Server Message Block): Dateifreigabe in Windows-Netzwerken.
    3306 - MySQL: Datenbankzugriff.
    3389 - RDP (Remote Desktop Protocol): Windows Remotedesktop.
    8080 - HTTP-Alt: Häufig für Proxy-Server oder Web-Caching


FTP username without acc = anonymous (download file = get)
